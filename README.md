# solo-adeventure

A web-based gamebook in the Choose Your Own Adventure style. Hand it a topic; Claude writes each page, FLUX illustrates, an Azure Neural voice reads it aloud, and a depth-displaced three.js plane gives every scene a subtle 3D parallax. Sign in with Google to keep your stories across sessions.

Live: https://solo-adeventure.ridopark.com

## Features

- **Narrative** per page generated by Claude Haiku 4.5 as a structured tool call (narrative + image prompt + 2-3 dilemmic choices + running summary). System prompt tuned for hooks, escalation, sensory rotation, and "yes-but / no-and" outcomes. Minimum 4 pages, max 8.
- **Illustration** per page via Together AI FLUX.1-schnell with fal.ai fallback. Single style descriptor per story for visual consistency.
- **Depth-based 2.5D parallax**: a Python sidecar runs Depth-Anything-V2-Small (ONNX, CPU) on each image to produce a 16-bit depth map. A three.js shader displaces a subdivided plane by that depth, a virtual camera orbits in a slow figure-eight, and the foreground visibly slides past the background. Auto-tuned per image (edge-aware displacement), with pause / show-original toggles.
- **Read-aloud narration**: a second Python sidecar wraps Microsoft Edge's free edge-tts endpoint (Azure Neural voices). Audio is synthesized on page load in the background and cached per page; the play button is zero-latency after that.
- **Story paging**: full history is kept in SQLite; the reader can Prev/Next through past pages or via Left/Right arrow keys.
- **Google OAuth SSO**: optional sign-in lets readers save stories across devices. Anonymous reads work for page 1; continuing requires sign-in.
- **Share button**: Web Share API on mobile, clipboard copy on desktop. Share URLs deep-link to page 1 so recipients read from the beginning.
- **Discord notifications** for every lifecycle event (topic submitted, page generated, image generated, choice made, visit started) attributed to the signed-in reader.
- **Content safety**: topic blocklist, PG-13 system prompt, image-prompt validator.

## Quick Start

Prerequisites: Go 1.25+, Node 22+, Python 3.11+, tmux.

```bash
cp .env.example .env     # fill in API keys (see Environment below)
./scripts/start.sh       # builds backend, bootstraps venvs, launches all 4 services in tmux
```

First boot creates two Python venvs (`services/tts/venv`, `services/depth/venv`) and downloads the Depth-Anything ONNX model (~100 MB) into `services/depth/hf-cache/`. Subsequent starts reuse everything.

Open http://localhost:3004.

Individual Makefile targets for backend or web alone:

```bash
make dev-backend
make dev-web
make test
```

## Architecture

**Hexagonal payoff.** Every external collaborator sits behind a port:

```go
type StoryProvider interface  { StartStyle(...); NextPage(...) }
type ImageProvider interface  { Generate(...) }
type TTSProvider interface    { Synthesize(...) }
type DepthProvider interface  { Estimate(...) }
type OAuthProvider interface  { AuthURL(...); Exchange(...) }
type Notifier interface       { Notify(...) }
```

Swapping Claude for another LLM, FLUX for another painter, edge-tts for OpenAI TTS, or Depth-Anything for a different depth model is one adapter file plus one line in `cmd/solo-adeventure-server/main.go`. The service layer and HTTP handlers never know the provider.

**Sidecars.** TTS and depth run as separate Python FastAPI services on localhost only. They're sidecars for two reasons: (1) the upstream libraries (`edge-tts`, `onnxruntime` with PyTorch-quality wheels) live in Python and re-implementing them in Go would be scope creep, and (2) process isolation means an upstream break (MS rotating the edge-tts endpoint, ONNX runtime failing) is recoverable with `pip install -U` + `systemctl restart` without touching the Go binary. Circuit-breaker env vars (`TTS_ENABLED=false`, `DEPTH_ENABLED=false`) disable either sidecar cleanly.

**Image proxy.** Together/fal CDNs don't send `Access-Control-Allow-Origin`, which breaks WebGL texture loading on the parallax plane. `GET /img?url=...` on the backend streams image bytes from a host allowlist so textures are same-origin.

**Depth-based 2.5D parallax.** Also known as the "3D photo" technique. The depth sidecar predicts a monocular depth map; the client displaces a subdivided plane's vertices along Z by the depth value, then orbits a camera around it. Vertex displacement creates real parallax (closer pixels shift further than distant ones), which the eye perceives as 3D motion from a single image. See `ParallaxIllustration.tsx` for the shader.

## Deployment

Single ARM64 VM on Oracle Cloud free tier running three systemd units (`solo-adeventure-server`, `solo-adeventure-tts`, `solo-adeventure-depth`), fronted by Caddy with auto Let's Encrypt. Frontend on Cloudflare Pages as a static export.

### One-time setup

**OCI VM** (Ubuntu 22.04+ ARM64, free tier):
- SSH access for `ubuntu` (passwordless sudo).
- Caddy installed with a reverse-proxy block for `solo-adeventure-api.example.com` -> `localhost:8084`.
- Ports 80 and 443 open.

**Cloudflare Pages**:
- Create a Pages project named `solo-adeventure` (Direct Upload).
- Generate a scoped API token with `Cloudflare Pages:Edit`.

**Google OAuth**:
- Create OAuth 2.0 credentials in Google Cloud Console.
- Add the production callback URL: `https://solo-adeventure-api.example.com/auth/google/callback`.

**GitHub Actions secrets** (Settings -> Secrets -> Actions):

| Secret                         | Value                                                      |
|--------------------------------|------------------------------------------------------------|
| `CLOUDFLARE_API_TOKEN`         | Cloudflare Pages API token                                 |
| `CLOUDFLARE_ACCOUNT_ID`        | Cloudflare account id                                      |
| `OCI_HOST`                     | public IP or DNS of the OCI VM                             |
| `OCI_USER`                     | `ubuntu`                                                   |
| `OCI_SSH_KEY`                  | private SSH key (PEM)                                      |
| `ANTHROPIC_API_KEY`            | Claude API key                                             |
| `TOGETHER_API_KEY`             | Together AI key (primary image)                            |

### CI/CD flow

Push to `main` -> `.github/workflows/ci.yml`:

1. **test**: `go test -race ./...` + ARM64 build verification.
2. **frontend-build**: `STATIC_EXPORT=1 npm run build` -> `apps/web/out/`.
3. **deploy-frontend**: `wrangler pages deploy out/ --project-name=solo-adeventure`.
4. **deploy-backend**:
   - Cross-compile ARM64 Go binary, scp to `/opt/solo-adeventure/`.
   - scp TTS sidecar files + systemd unit, create venv, pip install, enable + restart, smoke-test `/health`.
   - scp depth sidecar files + systemd unit, create venv, pip install (first boot downloads the ONNX model from HuggingFace), enable + restart, wait for `/health` up to 120s.
   - Write `.env` with all secrets, restart `solo-adeventure-server`, smoke-test `/health`.

### Route shape (why `/story?id=...`)

Cloudflare Pages static export does not support Next.js dynamic segments without `generateStaticParams`. The route is a single static page at `/story` that reads the id from `useSearchParams`. Behavior is identical; only the URL shape changed.
